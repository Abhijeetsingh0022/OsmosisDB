# Security Policy

## Supported Versions

Only the latest release of OsmosisDB is actively supported with security updates. We encourage all users to run the latest version to ensure they have the latest security patches.

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1.0 | :x:                |

---

## Reporting a Vulnerability

We take the security of OsmosisDB seriously. If you discover a security vulnerability in this project, please **do not** open a public issue on GitHub. Instead, report it privately to the maintainers.

### How to Report
1. Email your report to **security@osmosisdb.org** (or contact the repository owner directly).
2. Include the following details in your report:
   * A description of the vulnerability and its potential impact.
   * Step-by-step instructions or a proof-of-concept script to reproduce the issue.
   * Any potential mitigations or fixes you suggest.

### Our Response Process
* **Acknowledgment:** We will acknowledge receipt of your report within 48 hours.
* **Triage:** We will investigate and verify the vulnerability.
* **Fix & Release:** We will work on a patch and release a new minor/patch version as soon as possible.
* **Disclosure:** Once the fix is released, we will publish a security advisory detailing the issue and thanking you for your responsible disclosure.
